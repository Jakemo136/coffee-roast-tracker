Design decisions:
  • Single unified form — URL fetch and manual entry coexist. Fetch populates the same fields the user would fill manually, so there's no "mode switch" or lost context.
  • Async feedback — spinner + status text during fetch. User is free to start typing manually while waiting (fields aren't locked).
  • Populated fields — green-tinted border to indicate "auto-filled, review me." All fields remain editable.
  • Short name left blank — always user-entered, since this is their personal abbreviation for .klog matching.
  • Supplier Notes & Flavors always visible — user can enter these manually even without a URL fetch. After a successful fetch, Supplier Notes is pre-filled (editable textarea) and flavors gain a "from supplier" badge with pre-selected pills.
  • Save Bean disabled until Bean Name and Short Name are both filled.
  • Success toast — modal closes, toast slides up from bottom with "View bean →" link to the new bean detail page.
  • Error state — clear message, retry button, gentle nudge to enter manually. Doesn't block the form.
